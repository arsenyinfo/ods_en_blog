Before we turn to the seaborn and plotly methods, let’s discuss the simplest and often the most convenient way to visualize data from a pandas dataframe - by using the `plot()` function. As an example, we will make a plot of video games sales in different countries depending on the year. To begin with, let’s filter only the columns we need, then calculate the total sales by year, and call the `plot()` function on the resulting dataframe without any parameters.

The `plot()` function implementation in pandas is based on matplotlib library.


```python
df[[x for x in df.columns if 'Sales' in x] + 
   ['Year_of_Release']].groupby('Year_of_Release').sum().plot();
```


![svg](output_10_0.svg)


Using the `kind` parameter, you can change the type of the plot to e.g. a bar chart. Matplotlib allows quite flexible customization of plots. You can change almost everything on the chart, but you need to dig into the documentation and find the required parameters. For example, the parameter `rot` is responsible for the inclination angle of ticks for the x axis.


```python
df[[x for x in df.columns if 'Sales' in x] + 
   ['Year_of_Release']].groupby('Year_of_Release').sum().plot(kind='bar', rot=45);
```


![svg](output_12_0.svg)


## Seaborn

Now let's move on to the seaborn library. Seaborn is essentially a higher-level API based on the matplotlib library. Seaborn contains more adequate default settings for plotting. If you just add `import seaborn` to the code, the pictures will become much nicer. Also the library contains quite complex types of visualization that would require a large amount of code in matplotlib.

Let's get acquainted with the first such "complex" type of plots - a pair plot (scatter plot matrix). This visualization will help us to look at the relationship of different variables on a single picture.


```python
# pairplot works slow with svg format
%config InlineBackend.figure_format = 'png' 
sns.pairplot(df[['Global_Sales', 'Critic_Score', 'Critic_Count', 
                 'User_Score', 'User_Count']]);
```


![png](output_14_0.png)


It is also possible to plot distribution with `seaborn` dist plot. For example, let's look at the distribution of critics' ratings `Critic_Score`. By default the plot displays a histogram and [kernel density estimation](https://en.wikipedia.org/wiki/Kernel_density_estimation).


```python
%config InlineBackend.figure_format = 'svg' 
sns.distplot(df.Critic_Score);
```


![svg](output_16_0.svg)


In order to look more closer at the relationship between two numerical variables, you can also use a __`joint_plot`__  - a hybrid of a `scatter plot` and a `histogram`. Let's see how the `Critic_Score` and the `User_Score` are related.


```python
sns.jointplot(x='Critic_Score', y='User_Score', 
              data=df, kind='scatter');
```


![svg](output_18_0.svg)


Another useful type of plots is a __`box plot`__. Let's compare critics' ratings for games on the top 5 biggest gaming platforms.


```python
top_platforms = df.Platform.value_counts().sort_values(ascending = False).head(5).index.values
sns.boxplot(y="Platform", x="Critic_Score", 
            data=df[df.Platform.isin(top_platforms)], orient="h");
```


![svg](output_20_0.svg)


I think it's worth discussing a bit more about how to interpret a `box plot`. `Box plot` consists of a box (that's why it is called a `box plot`), whiskers and dots. The box shows the interquartile spread of the distribution, that is, respectively 25% (`Q1`) and 75% (`Q3`) percentiles. The line inside the box indicates the median of the distribution. Now that we’ve sorted out the box, let's move on to the whiskers. The whiskers represent the entire scatter of the points except the outliers. That is, the minimum and maximum values that fall within the interval `(Q1 - 1.5*IQR, Q3 + 1.5*IQR)`, where `IQR = Q3 - Q1` is interquartile range. Points on the graph indicate outliers - those values that do not fit into the range of values bounded by the whiskers.

One more type of plots (the last one in this article) is the `heat map`. Heat map allows to look at the distribution of some numerical variable over two categorical ones. Let’s visualize the total sales of games by genre and gaming platform.

## Plotly

We examined the visualizations based on the `matplotlib` library. However, this is not the only option for plotting in `python`. Let’s also get acquainted with the __`plotly`__ library. `Plotly` is an open-source library that allows you to build interactive graphics/plots in jupyter notebook without having to bury oneself into javascript.

The beauty of interactive plots is that you can see the exact numerical value by hovering the mouse, hide uninteresting items in the visualization, zoom in a certain part of the plot, etc.

Before we start, let’s import all the necessary modules and initialize `plotly` with the `init_notebook_mode()` command.


```python
from plotly.offline import download_plotlyjs, init_notebook_mode, plot, iplot
import plotly
import plotly.graph_objs as go

init_notebook_mode(connected=True)
```

First of all, let’s build a __`line plot`__ with the dynamics of the number of games released and their sales by year.


```python
years_df = df.groupby('Year_of_Release')[['Global_Sales']].sum().join(
    df.groupby('Year_of_Release')[['Name']].count())
years_df.columns = ['Global_Sales', 'Number_of_Games']
```

`plotly` makes visualization of the `Figure` object, which consists of data (an array of lines called `traces` in the library) and the design, or style, which the `layout` object is responsible for. In simple cases you can just call the `iplot()` function from the `traces` array.


```python
trace0 = go.Scatter(
    x=years_df.index,
    y=years_df.Global_Sales,
    name='Global Sales'
)

trace1 = go.Scatter(
    x=years_df.index,
    y=years_df.Number_of_Games,
    name='Number of games released'
)

data = [trace0, trace1]
layout = {'title': 'Statistics of video games'}

fig = go.Figure(data=data, layout=layout)

iplot(fig, show_link=False)
```

Let’s construct a t-SNE representation of the same churn data. The name of the method is complex - t-distributed Stohastic Neighbor Embedding. Math is also impressive (and we’ll not delve into it). But the basic idea is a breeze: find such a representation of a multidimensional feature space onto a plane (or 3D, but almost always 2D is chosen) so that points that were far apart in the initial space also appeared far apart on the plane, and those that were close also remained close to each other. That is, neighbor embedding is a kind of search for a new data representation that preserves neighborhood.

A few details: let’s throw away the State and Churn features and convert the binary Yes/No features to numbers ( [`pandas.Series.map`](http://pandas.pydata.org/pandas-docs/stable/generated/pandas.Series.map.html)). We also need to scale the data - subtract mean from each variable and divide it by its standard deviation - this is all done by `StandardScaler`.


```python
from sklearn.manifold import TSNE
from sklearn.preprocessing import StandardScaler
```

Finally, we’ll note some disadvantages of t-SNE (yes, it’s better to write a separate article on this method):
- great computational complexity. This sklearn implementation will most likely not help you in a real task. On large samples, it’s better to look at [Multicore-TSNE](https://github.com/DmitryUlyanov/Multicore-TSNE);
- the plot can change a great deal when changing a random seed, and this complicates interpretation. [Here](http://distill.pub/2016/misread-tsne/) is a good tutorial on t-SNE. But in general you shouldn’t make any far-reaching conclusions based on such pictures, it’s better not to guess. Sometimes some findings strike the eye and are confirmed later in the study, but it does not happen very often.
